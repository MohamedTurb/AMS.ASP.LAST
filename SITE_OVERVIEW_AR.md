# ملخص الموقع وصفحة بصفحة

هذا المشروع هو نظام لإدارة المساعدات، يغطّي تسجيل المستفيدين، طلبات المساعدة، الفروع، المنظمات، المشاريع، تصنيفات المساعدات، الإشعارات، التقارير، وإدارة المستخدمين والصلاحيات.

## الموقع بشكل عام

- تسجيل الدخول والتسجيل للمستخدمين الجدد.
- لوحة رئيسية تختلف حسب الدور: مدير النظام يرى مؤشرات عامة، بينما المستفيد يرى بياناته وطلباته فقط.
- إدارة طلبات المساعدة مع البحث، التصفية، المراجعة، القبول، الرفض، وإضافة المساعدة بعد الموافقة.
- إدارة المستفيدين والفروع والمنظمات والمشاريع وتصنيفات المساعدات.
- عرض الإشعارات وتحديدها كمقروءة.
- تقارير وتحليلات، مع تصدير بعض البيانات إلى CSV وملفات Excel وPDF.

## الصفحات الرئيسية

| الصفحة | ماذا تفعل |
|---|---|
| Home/Index | الصفحة الرئيسية بعد الدخول. تعرض لوحة تحكم عامة للإداريين، أو لوحة خاصة بالمستفيد تعرض عدد طلباته وحالتها والطلبات الحديثة. |
| Home/BeneficiaryDashboard | عرض بيانات المستفيد فقط داخل الصفحة الرئيسية عند تسجيل دخوله بحساب مستفيد. |
| Home/Privacy | صفحة سياسة الخصوصية الثابتة. |
| Account/Login | تسجيل الدخول للمستخدمين الحاليين. |
| Account/Register | إنشاء حساب جديد، ويُسند الحساب افتراضيًا إلى دور Beneficiary. |
| Account/Index | عرض قائمة المستخدمين، ومتاح للإدمن فقط. |
| Account/Create | إنشاء مستخدم جديد من لوحة الإدارة. |
| Account/Edit | تعديل بيانات مستخدم وتغيير دوره أو فرعه من لوحة الإدارة. |
| Account/Delete | حذف مستخدم من لوحة الإدارة. |

## ملاحظات مهمة عن الصلاحيات

- بعض الصفحات متاحة للجميع بعد تسجيل الدخول فقط.
- بعض الصفحات مخصصة للإدمن فقط مثل إدارة المستخدمين والفروع.
- مدير الفرع يرى البيانات الخاصة بفرعه فقط في أكثر من صفحة.
- المستفيد يرى بياناته وطلباته فقط في اللوحة الخاصة به.

## صفحات أو أوامر غير واجهة المستخدم

هذا الملف يركز على صفحات الواجهة. يوجد أيضًا API وعمليات خلفية مثل تسجيل الدخول بالتوكن، تحويل المستفيد بين الفروع، وإدارة بيانات عبر endpoints، لكنها ليست صفحات UI مباشرة.
